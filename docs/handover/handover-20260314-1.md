## Open Issues

### CRITICAL: Scheduling Layer Needs Redesign

Cowork is **not fit for purpose** as sole scheduling backbone. Two severe limitations:

1. **No programmatic task creation** — Tasks only created via UI or `/schedule` paste. Gaia cannot self-provision or update its own tasks.
2. **Requires machine awake** — Tasks only fire while Claude Desktop is open and machine is awake. Missed runs get one catch-up (most recent only, within 7 days). Unreliable for daily rhythm.
